## 📋 Reglas de Orquestación

### 1. Token Optimization (Eficiencia de Tokens)
- Cada SKILL se activa **solo cuando es necesaria** para la tarea actual
- Las SKILLs son modulares y no se cargan simultáneamente
- Referencia a assets por path relativo, no por contenido completo

### 2. Design Pattern Commitment
- **Observer Pattern** → Animaciones basadas en Intersection Observer
- **Module Pattern** → JS separado en módulos independientes
- **Component Pattern** → CSS con BEM methodology
- **Strategy Pattern** → Pricing tabs con estrategia intercambiable

### 3. Quality Standards
- HTML5 semántico y accesible (WCAG 2.1 AA)
- CSS Variables para todo el Design System
- Vanilla JS (sin frameworks) para máximo rendimiento
- Mobile-first responsive design
- Core Web Vitals optimizados (LCP < 2.5s)

### 4. File Separation Protocol
- HTML: Estructura semántica pura
- CSS: Estilos separados (global / componentes / animaciones)
- JS: Módulos separados por responsabilidad
- Assets: Organizados por tipo en `/assets/`
